в проекте. Пусть и от gitlab/master, но она (наверное) общая

Но перед тем как переключиться IDE будет просить сохранитиь/закомитить изменения

Чтобы не мучаться постоянно с подключением через HTTPS создадим ключи для подключения черз механизм SSH

Можем прочесть инструкцию как создавать ssh ключи

В GitLab к примеру можем увидеть следующие виды SSH ключей которые можем сгенерировать:

Сейчас же просто создадим Ключ для работы/доступа к GitHub и GitLab
Для GitHub вначале нужно открыть терминал git bash, затем проверим какие ключи SSH имееются, вбив команду - ls ~/.ssh
и увидим список файлов - id_rsa (приватный) id_rsa.pub (публичный) и сами ключи, которые по сути хранятся на ПК
по адресу - C:\Users\7prof\.ssh)

Далее создадим вид ключа первый из списка выше - ED25519 (preferred), командой - ssh-keygen -t ed25519 -C "<comment>"
Вначале сгенерируем без комментария. Просто вобьем в GitBash - ssh-keygen -t ed25519 Далее поочередно 3 раза необходимо
Нажать Enter, в первом случае ничего не вводим, во втором случае попросит Фразу, можно ввести 123 которую будет
необходимо ввести при подключении к удаленному репозитарию с этим SSH ключем. В итоге получим сгенерированные 2 ключа
- id_ed25519 и id_ed25519.pub

К прмеру будем работать с публичным, открываем его в редакторе и копируем все что до - 7prof@Al
То есть - ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAILlP2sdF9XUhb5H2Cw61iAHeSIi+i3BWUiD2UZ/gwxrS
Далее в GitHub нажимаем на логотип профиля, выбираем - Settings, далее - SSH and GPC keys, далее - New SSH key
Где в открывшимся окне предложат дать название ключу (к примеру - key) и ввести сам ключ, что и сделаем
После чего необходимо подтвердить паролем от GitHub

Теперь необходимо удалить все что связано с предыдущими удаленными подключениями через HTTPS (remotes)
Мы создавали по два вида с названиями - github и gitlab Вспомним что вбив команду - git remote -v,
мы получали список:

Эти удаленные подключения и список необходимо удалить, так как больше нам не пригодятся. Сделаем это

Теперь создадим новый remotes для SSH для GitHub. Просто вбиваем команду - git remote add github с добавлением
строки которую возьмем там же что и для HTTPS ранее - Code -> SSH - > git@github.com:1devorg/git_laravel.git
В нужном проекте - git_laravel. Итоговая команда для создания удаленного подключения по SSH для GitHub выглядит так -
git remote add github git@github.com:1devorg/git_laravel.git Где github это название подключения. Можем проверить
через ту же команду что и выше - git remote -v и получим список новых подключений:

github  git@github.com:1devorg/git_laravel.git (fetch)
github  git@github.com:1devorg/git_laravel.git (push)

Тоже самое проделаем для GitLub - git remote add gitlab git@gitlab.com:1devorg/git_laravel.git

Далее для соединения с удаленными репозитариями выберем Fetch в правом ниженм углу IDE, или через терминал
Либо отправим какой-то Комит на него. При этом он попросит подтвердить что доверяем новому подключени и попросит
ввести Фразу, которую мы создали при генерации ключа (выше) - 123
